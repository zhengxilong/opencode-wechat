# 排障指南

[English](./troubleshooting.md) | [简体中文](./troubleshooting.zh-CN.md)

本文档汇总 `opencode-wechat` 常见的安装和运行问题。

## 1. 找不到 `opencode` 命令

### 现象

Bridge 启动失败，或者无法创建 OpenCode Session。

### 排查点

- 确认在同一个 shell 中执行 `opencode --help` 可以成功。
- 确认 Bridge 运行时使用的 `PATH` 中包含 OpenCode 二进制。
- 确认配置中的 `opencode.command` 值正确。

## 2. 配置校验失败

### 现象

启动时出现配置解析或校验错误。

### 排查点

- 确认配置文件名为 `wechat-channel.json`。
- 确认 JSON 格式合法。
- 确认必要字段已经填写。
- 确认配置中的项目目录真实存在。

## 3. 微信登录失败或登录态失效

### 现象

二维码登录无法完成，或者 Bridge 重启后登录态丢失。

### 排查点

- 确认运行状态目录可写。
- 确认登录态是否已经持久化到配置的状态目录中。
- 确认你当前使用的微信接入方式在当前环境里仍然有效。

## 4. 能收到消息但没有返回结果

### 现象

微信消息已经发送成功，但没有收到任何回复。

### 排查点

- 确认目标项目映射是否正确。
- 确认 OpenCode 能访问配置的项目目录。
- 确认 Session 创建过程没有失败。
- 确认 Bridge 日志中能看到完整的 OpenCode 响应链路。

## 5. 回复过长或被截断

### 现象

微信返回内容被截断，或者结果不完整。

### 排查点

- 检查 `max_reply_chars` 的配置值。
- 如有需要，补充或优化消息拆分逻辑。
- 先用更短的 Prompt 做一次对照测试。

## 6. 重启后行为不一致

### 现象

Bridge 重启后，Session 复用或登录态恢复结果与预期不一致。

### 排查点

- 明确哪些运行状态应当被持久化。
- 确认状态文件已经成功写入。
- 确认运行状态目录没有发生变化。

## 7. 出现不该直接执行的高风险操作

### 现象

Bridge 看起来允许执行本应经过确认的高风险操作。

### 排查点

- 确认 `require_approval_for_risky_actions` 已启用。
- 检查高风险动作的分类规则是否覆盖当前场景。
- 在更广泛使用前，显式测试审批确认流程。

## 8. 提交 Issue 前建议收集的信息

在报告问题前，建议尽量收集：

- 你执行的命令
- 去掉敏感信息后的配置结构
- 完整的错误信息
- 相关日志片段
- 重启后是否仍然可以稳定复现
