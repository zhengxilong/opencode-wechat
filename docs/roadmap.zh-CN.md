# Roadmap

[English](./roadmap.md) | [简体中文](./roadmap.zh-CN.md)

本文档描述 `opencode-wechat` 作为独立开源项目，后续比较合理的演进路径。

## Phase 0：仓库基础建设

目标：让仓库本身清晰、可理解、可公开发布。

重点包括：

- 完成 README 和双语文档
- 清理仓库结构
- 提供示例配置
- 提供示例项目
- 提供发布 checklist 和面向公开用户的说明

## Phase 1：源码方式稳定可用

目标：让个人开发者可以稳定地从源码运行 Bridge。

重点包括：

- 清晰的独立配置加载逻辑
- 更可靠的启动流程
- 更稳定的本地状态持久化
- 更好的运行时诊断能力
- 更清晰的错误提示

## Phase 2：真实场景联调强化

目标：让真实微信到 OpenCode 的链路更稳定。

重点包括：

- 重复登录与重启恢复验证
- 更稳的消息路由行为
- 更稳的 Session 复用逻辑
- 审批确认流程细化
- 更完善的运维排障指南

## Phase 3：测试与质量建设

目标：降低回归风险，提高协作者信心。

重点包括：

- 单元测试
- 集成测试
- smoke 测试
- 文档化测试场景
- 适合 CI 的校验策略

## Phase 4：独立打包与安装体验

目标：让安装过程更轻、更顺滑。

重点包括：

- 独立的 `opencode-wechat` 打包
- 构建脚本增强
- 发布产物策略
- 二进制分发方案评估

## Phase 5：安全与可观测性强化

目标：提升长期运行时的可控性与可信度。

重点包括：

- 更安全的动作策略
- 更清晰的审批边界
- 更强的日志与诊断能力
- 可选的指标与审计增强

## Phase 6：社区化准备

目标：让项目更容易被更广泛的开发者使用和参与。

重点包括：

- issue template
- 贡献指南
- 更清晰的文档结构
- 更明确的兼容性说明
- 面向不同部署模式的示例

## 短期优先事项

接下来最有价值的事情包括：

1. 持续进行真实微信与 OpenCode 联调
2. 补上自动化测试
3. 推进独立二进制打包路径
4. 继续加强安全、状态管理和诊断能力
