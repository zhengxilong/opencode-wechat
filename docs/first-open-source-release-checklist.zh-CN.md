# 首次开源发布 Checklist

[English](./first-open-source-release-checklist.md) | [简体中文](./first-open-source-release-checklist.zh-CN.md)

在将 `opencode-wechat` 作为独立 GitHub 开源项目正式发布前，建议先逐项确认以下内容。

## 1. 仓库基础信息

- [ ] 创建独立 GitHub 仓库
- [ ] 确认最终仓库名称
- [ ] 在 GitHub 中补充项目简介和 topics
- [ ] 添加公开许可证文件
- [ ] 添加 `.gitignore`
- [ ] 确认默认分支策略

## 2. 文档准备

- [ ] 完成 `README.md`
- [ ] 完成 `README.zh-CN.md`
- [ ] 在 README 中加入语言切换链接
- [ ] 添加 `docs/quick-start.md`
- [ ] 添加配置说明文档
- [ ] 添加排障文档
- [ ] 添加安全与风险说明
- [ ] 添加架构说明文档
- [ ] 添加 roadmap 文档

## 3. 配置与示例

- [ ] 提供 `wechat-channel.example.json`
- [ ] 清理示例中的本机绝对路径
- [ ] 在 `examples/` 下提供最小 demo 项目
- [ ] 确认示例配置可以直接复制后修改使用

## 4. 代码可发布性

- [ ] 去掉项目内部专用的硬编码路径
- [ ] 去掉公司或环境专用假设
- [ ] 确认运行状态目录可配置
- [ ] 确认启动报错信息足够清晰且可操作
- [ ] 确认 Bridge 重启行为可接受
- [ ] 确认登录态持久化行为已有文档说明

## 5. 测试

- [ ] 至少补充基础单元测试
- [ ] 至少补充一个集成测试或 smoke 测试
- [ ] 验证 fresh clone 后可以成功启动
- [ ] 验证文档中的 quick-start 流程与实际一致
- [ ] 验证示例配置替换路径后可运行

## 6. 打包与开发者体验

- [ ] 确认 `package.json` 元数据正确
- [ ] 确认 `tsconfig.json` 适合独立仓库使用
- [ ] 添加 `doctor` 命令
- [ ] 添加后续独立打包所需的 build 脚本
- [ ] 决定第一版是仅源码发布还是附带二进制辅助
- [ ] 明确支持的 Bun 和 Node 要求

## 7. 安全与合规

- [ ] 检查微信登录相关账号安全说明
- [ ] 检查高风险操作确认的默认策略
- [ ] 明确项目的使用边界
- [ ] 如有必要，明确说明仅建议在受控环境使用
- [ ] 清理仓库中的敏感日志、Token 和私有地址

## 8. 发布管理

- [ ] 确定首个版本号，例如 `v0.1.0`
- [ ] 编写 release notes
- [ ] 准备截图或 GIF 演示
- [ ] 如有需要，补充 issue template
- [ ] 如果希望社区协作，补充贡献指南
- [ ] 打上首个公开版本标签

## 9. 发布后计划

- [ ] 准备后续 2 到 3 个里程碑的简短 roadmap
- [ ] 决定如何处理 issue 和支持请求
- [ ] 收集第一批用户的安装反馈
- [ ] 根据反馈优先补测试、二进制打包和文档
